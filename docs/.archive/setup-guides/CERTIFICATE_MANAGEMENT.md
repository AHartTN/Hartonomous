# SSL Certificate Management

## Overview

This deployment uses a **centralized certificate management** approach where Let's Encrypt certificates are managed on the router (192.168.1.1) using ACME protocol, and then synced to deployed servers.

## Architecture

```
Router (192.168.1.1:22122)
├─ acme.sh manages Let's Encrypt certificates
├─ Auto-renews before expiry
└─ Stores in /etc/acme/ and /etc/ssl/acme/

        ↓ SSH (SCP sync via systemd timer)

Deployed Servers (hart-server, HART-DESKTOP)
├─ Certificates: /etc/ssl/certs/, /etc/ssl/private/
├─ nginx uses synced certificates
└─ Daily automated sync at 3 AM
```

**Benefits:**
- Single source of truth for certificates
- Router handles ACME renewals automatically
- Secure SSH key authentication
- Automated daily rotation via systemd timer

## Router Certificate Structure

### Managed by acme.sh

```
/etc/acme/hartonomous.com_ecc/
├─ hartonomous.com.cer      # End-entity certificate
├─ hartonomous.com.key      # Private key (ECC P-256)
├─ fullchain.cer            # Full chain (cert + intermediates)
├─ ca.cer                   # CA certificate chain
└─ combined.cer             # Combined cert + key

/etc/ssl/acme/               # Symlinks for easy access
├─ hartonomous.com.crt → /etc/acme/hartonomous.com_ecc/hartonomous.com.cer
├─ hartonomous.com.key → /etc/acme/hartonomous.com_ecc/hartonomous.com.key
├─ hartonomous.com.fullchain.crt → /etc/acme/hartonomous.com_ecc/fullchain.cer
└─ hartonomous.com.chain.crt → /etc/acme/hartonomous.com_ecc/ca.cer
```

**Certificate Details:**
- Domain: hartonomous.com (with wildcard *.hartonomous.com)
- CA: Let's Encrypt E8 intermediate (ECDSA)
- Validity: 90 days, auto-renewed ~30 days before expiry

## Deployed Server Setup

### Certificate Sync Script

**Location:** `scripts/setup-sync-certificates.sh`

**Features:**
- Idempotent - safe to run multiple times
- SSH key authentication to router (port 22122)
- Certificate validation (format, key match, expiration)
- Only updates if certificate changed
- Nginx reload after certificate update
- Systemd timer for automated daily sync

### Installation


## SSH Key Authentication Setup

### 1. Generate SSH Key (First Time)

```bash
# Auto-generated by setup script, or manually:
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519 -N ""
```

### 2. Copy Public Key to Router

```bash
# Using ssh-copy-id:
ssh-copy-id -p 22122 root@192.168.1.1

# Or manually:
cat ~/.ssh/id_ed25519.pub
# Add to /root/.ssh/authorized_keys on router
```

### 3. Test Connection

```bash
ssh -p 22122 root@192.168.1.1 "echo 'Connection successful'"
```

## Automated Certificate Rotation

### Systemd Timer

**Service:** `/etc/systemd/system/cert-sync.service`  
**Timer:** `/etc/systemd/system/cert-sync.timer`

- Runs daily at 3 AM
- Also runs 5 minutes after boot
- Randomized delay up to 1 hour (avoid thundering herd)

### Management Commands

```bash
# Enable and start timer
sudo systemctl enable --now cert-sync.timer

# Check timer status
systemctl status cert-sync.timer
systemctl list-timers cert-sync.timer

# Check last sync logs
journalctl -u cert-sync.service -n 50

# Manual sync
sudo /usr/local/bin/setup-sync-certificates.sh
```

## nginx Configuration

nginx uses synced certificates:

```nginx
server {
    listen 443 ssl http2;
    server_name hartonomous.com *.hartonomous.com;
    
    ssl_certificate     /etc/ssl/certs/hartonomous.com.fullchain.crt;
    ssl_certificate_key /etc/ssl/private/hartonomous.com.key;
    
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    
    # ... rest of configuration
}
```

## Certificate Lifecycle

1. **Router auto-renewal** (30 days before expiry):
   - acme.sh renews certificate with Let's Encrypt
   - New certificate stored in `/etc/acme/hartonomous.com_ecc/`

2. **Automatic sync** (daily at 3 AM):
   - Systemd timer triggers sync
   - Detects certificate changed
   - Downloads from router via SCP
   - Validates certificate
   - Installs to `/etc/ssl/`
   - Reloads nginx

3. **Monitoring:**
   ```bash
   # Check certificate expiration
   echo | openssl s_client -connect hartonomous.com:443 2>/dev/null | \
     openssl x509 -noout -dates
   
   # Check sync logs
   journalctl -u cert-sync.service --since "24 hours ago"
   ```

## Troubleshooting

### Certificate Not Syncing

```bash
# Test SSH connection
ssh -p 22122 root@192.168.1.1 "ls -la /etc/ssl/acme/"

# Check timer
systemctl status cert-sync.timer

# Manual sync with verbose output
sudo bash -x /usr/local/bin/setup-sync-certificates.sh

# Check logs
journalctl -u cert-sync.service -n 100
```

### nginx Not Reloading

```bash
# Test nginx configuration
sudo nginx -t

# Check nginx logs
sudo journalctl -u nginx -n 50

# Manual reload
sudo systemctl reload nginx
```

### SSH Key Issues

```bash
# Verify SSH key exists
ls -la ~/.ssh/id_ed25519*

# Test authentication
ssh -vvv -p 22122 root@192.168.1.1 "echo test"

# Re-copy key
ssh-copy-id -p 22122 root@192.168.1.1
```

## Troubleshooting

### Certificate Not Found in Pipeline

**Cause:** Service connection not configured or lacks permissions

**Fix:**
```bash
# Grant Key Vault access to service principal
az keyvault set-policy \
    --name hartonomous-kv \
    --spn <service-principal-id> \
    --certificate-permissions get list \
    --secret-permissions get list
```

### Certificate Expired

**Cause:** Auto-rotation failed or was disabled

**Fix:**
```bash
# Force rotation
./scripts/setup-certificate.ps1
```

### Build Agent Can't Install Certificate

**Cause:** Certificate format or permissions issue

**Fix:** Check pipeline logs - certificate should download as base64. Verify Key Vault access policy includes `get` and `list` for secrets.
